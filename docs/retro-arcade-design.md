# 童年怀旧游戏合集 App 设计稿 v0.1

## 1. 项目定位

这是一个以“按键机时代小游戏回忆”为核心的安卓应用。

目标不是做现代手游那种复杂联机、皮肤、广告驱动的产品，而是做一个：

- 打开就能玩
- 规则简单直接
- 画面复古克制
- 尽量离线可用
- 以后适合开源维护

首发版本建议先只做好一个游戏：`经典贪吃蛇`。

原因：

- 它最能代表按键机童年记忆
- 规则清晰，适合先把整个 App 框架搭起来
- 后续新游戏可以复用同一套主页、设置、排行榜、音效、振动和渲染思路

## 2. 产品方向

### App 形态

建议把它做成“怀旧游戏盒子”，而不是单独一个贪吃蛇 App。

这样有两个好处：

- 现在先上线贪吃蛇，后面加别的游戏不需要推倒重来
- GitHub 开源时，项目故事会更完整，也更容易持续更新

### 首发体验

用户第一次打开应用，看到的是一个复古主页：

- 应用标题
- 游戏列表
- 当前只有 `经典贪吃蛇`
- 一个“设置/关于”入口

点进贪吃蛇后，进入游戏准备页：

- 开始游戏
- 难度选择：慢 / 中 / 快
- 操作方式：方向键 / 滑动
- 主题风格：经典绿屏 / 深色夜光
- 历史最高分

进入游戏后：

- 顶部显示当前分数
- 中间是网格地图
- 底部是复古风方向按钮，或者启用滑动控制
- 支持暂停 / 重新开始 / 返回主页

游戏结束后：

- 显示本局得分
- 是否刷新纪录
- 再来一局
- 返回列表

## 3. 视觉风格

建议整体不要做得太花，而是明确模仿“老式设备上的小游戏感”。

### 推荐视觉关键词

- 像素感
- 低饱和
- 单色或双色主视觉
- 简洁边框
- 轻微电子屏氛围

### 主题建议

先做 2 套主题就够：

1. `经典绿屏`
   - 背景偏深绿
   - 食物和蛇身用亮绿
   - 像老式 LCD

2. `夜光黑底`
   - 黑底 + 青绿线条
   - 更适合现代手机屏幕

### 字体与 UI 风格

建议标题和分数使用有“像素味道”的字体风格，但不要过度影响中文可读性。

界面结构尽量方正：

- 卡片不要太圆
- 按钮边缘明确
- 动画轻量

## 4. 技术框架建议

## 总体技术选型

建议第一版使用：

- `Kotlin`
- `Jetpack Compose`
- `Navigation Compose`
- `ViewModel`
- `DataStore`

暂时不建议一开始就引入很重的游戏引擎，比如 Unity 或 LibGDX。

原因：

- 我们的目标是“轻量、复古、规则简单”的小游戏
- 贪吃蛇本质是网格状态更新，Compose + Canvas 足够完成
- Android Studio 原生开发链更顺，后续开源也更容易读懂

## 为什么用 Compose

Compose 很适合这类项目：

- 主页、设置页、排行榜页可以直接用声明式 UI 写
- 贪吃蛇地图可以用 `Canvas` 渲染
- 状态管理清晰，适合逐步扩展成小游戏合集

## 数据层建议

第一版只需要轻量存储，不必马上上数据库。

建议用 `DataStore` 存这些内容：

- 最高分
- 上次选择的难度
- 上次选择的主题
- 控制方式
- 音效开关
- 振动开关

如果以后要做：

- 多游戏战绩历史
- 成就系统
- 对局记录

再考虑引入 `Room`。

## 5. 推荐代码结构

第一阶段建议仍然保持单模块 `app`，但包结构按功能拆清楚。

推荐目录思路：

```text
app/src/main/java/你的包名/
  app/
    MainActivity.kt
    RetroArcadeApp.kt
    navigation/
  core/
    model/
    ui/
    theme/
    datastore/
    sound/
  feature/
    home/
    settings/
    snake/
      ui/
      domain/
      model/
      engine/
```

### 各层职责

`feature/home`

- 游戏列表页
- 进入各小游戏

`feature/settings`

- 音效、振动、主题

`feature/snake/model`

- 坐标
- 方向
- 游戏配置
- 游戏状态

`feature/snake/engine`

- 贪吃蛇核心规则
- 每一帧更新逻辑
- 碰撞判断
- 食物生成
- 得分计算

`feature/snake/ui`

- 游戏画面
- 控制按钮
- 暂停/结束弹窗

这样做的好处是：

- 游戏规则和界面分离
- 后面新增“俄罗斯方块/打砖块/赛车避障”时结构可复用
- 单元测试可以直接测 `engine`

## 6. 贪吃蛇核心设计

### 核心玩法

采用最经典规则：

- 蛇不断向当前方向移动
- 吃到食物后身体变长
- 撞墙或撞到自己则游戏结束
- 分数随食物数量增加

### 建议配置

第一版建议支持：

- 难度：慢 / 中 / 快
- 控制：方向键 / 手势滑动
- 主题：绿屏 / 夜光
- 暂停 / 继续
- 历史最高分

### 第一版先不做

为了控制复杂度，先不做这些：

- 联机
- 广告
- 道具系统
- 地图障碍物
- 穿墙模式
- 成就系统
- 多存档

这些都可以作为后续版本再加。

### 画面建议

游戏区域建议做成规则网格。

例如：

- 竖屏模式下自动计算可容纳的行列数
- 每个格子保持正方形
- 蛇头可以和身体略有区别
- 食物用闪烁点或方块表示

### 控制建议

优先做两种控制方式：

1. `方向键模式`
   - 最有按键机味道
   - 屏幕底部四方向按钮

2. `滑动模式`
   - 更适合现代用户
   - 在游戏区域滑动切换方向

默认建议使用 `方向键模式`。

## 7. 贪吃蛇状态模型建议

这部分是后面真正写代码时的核心。

可以围绕下面几个数据结构展开：

```kotlin
data class Cell(val x: Int, val y: Int)

enum class Direction {
    UP, DOWN, LEFT, RIGHT
}

data class SnakeConfig(
    val rows: Int,
    val cols: Int,
    val tickMillis: Long
)

data class SnakeGameState(
    val snake: List<Cell>,
    val direction: Direction,
    val pendingDirection: Direction?,
    val food: Cell,
    val score: Int,
    val isRunning: Boolean,
    val isPaused: Boolean,
    val isGameOver: Boolean
)
```

核心更新逻辑可以做成一个纯逻辑引擎，例如：

- `startNewGame(config)`
- `enqueueDirection(direction)`
- `tick()`
- `pause()`
- `resume()`

这样后面 UI 只负责显示状态和发送输入，逻辑会很清楚。

## 9. 后续可扩展的游戏

当贪吃蛇稳定后，可以按复杂度逐步扩充：

- `方块消除/下落方块`
- `打砖块`
- `赛车避障`
- `飞机大战极简版`
- `推箱子`

建议新增游戏时都遵守同一个原则：

- 规则简单
- 单机离线
- 一分钟内可开始
- 三分钟内能获得反馈

## 10. 开发顺序建议

建议我们按下面顺序推进，而不是一次性全做：

1. 建立 Android 项目骨架
2. 先做主页 + 导航 + 设置存储
3. 实现贪吃蛇纯逻辑引擎
4. 接入 Compose Canvas 做渲染
5. 加方向键和滑动控制
6. 加音效、振动、暂停、结算弹窗
7. 打磨主题和像素风 UI
8. 补 README、截图、开源说明

## 11. 测试建议

至少做这几类验证：

- 不同屏幕尺寸下，网格是否正常显示
- 快速连续转向时，是否出现反向穿模 bug
- 食物是否会生成在蛇身体上
- 暂停恢复后，速度是否异常
- 横竖屏切换后，状态是否符合预期

如果后面进入实现阶段，建议给贪吃蛇引擎补一些单元测试。

## 12. 我建议的第一版结论

如果由我来带这个项目往下走，我建议第一步定成：

`做一个可扩展的怀旧游戏盒子，首发只上线经典贪吃蛇。`

技术上：

- 用 Kotlin + Jetpack Compose
- 用单 Activity + Compose Navigation
- 用 DataStore 存设置和最高分
- 用独立的 Snake Engine 管理规则
- 用 Compose Canvas 画游戏地图

产品上：

- 先把“复古感”和“简单直接”做出来
- 不急着做多游戏
- 不急着做复杂特效
- 先追求好玩、稳定、开源后别人也看得懂
